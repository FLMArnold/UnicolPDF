# v0.2 History & Auto-save Design

## Scope

Implement SPECv0.2 features: history/undo/redo, history limit setting, auto-save with crash recovery, and export path fix.

## Core Architecture

### 1. History Manager (`app/history_manager.py`) — new file

```python
HistoryManager:
    _entries: list[BytesIO]       # PDF snapshots, oldest=[0], newest=[-1]
    _descriptions: list[str]      # parallel, human-readable operation name
    _cursor: int                  # index of current (#0) state
    _max_steps: int               # configurable, default 20

    add_snapshot(buf, desc)       # truncates future, appends, enforces limit
    undo() -> BytesIO | None      # cursor -= 1, return snapshot
    redo() -> BytesIO | None      # cursor += 1, return snapshot
    navigate(index) -> BytesIO    # jump to any entry, update cursor
    can_undo() -> bool
    can_redo() -> bool
    entry_count() -> int
    get_entries() -> list         # for UI: (desc, is_current, is_future)
```

- `cursor == len(entries) - 1` means at newest (normal)
- `cursor < len(entries) - 1` means undid some steps
- Display numbering: #0=cursor, #N=cursor-N for older, #-N=cursor+N for newer
- On max_steps exceeded: remove entries[0] (oldest), shift cursor left by 1

### 2. History Panel Tab (`app/history_panel.py`) — new file

QWidget with QListWidget showing entries:
- Entries before cursor: bright text (undo direction)
- Entry at cursor: highlighted (current #0)
- Entries after cursor: gray text (redo direction)
- Click item → navigate to that state
- Display format: "[操作描述] (#N)"

### 3. Integration in `app/main_window.py`

- HistoryManager instance, initialized on PDF open
- After every successful operation, call `_save_history_snapshot(desc)`
- Ctrl+Z → undo, Ctrl+Y → redo
- Auto-save counter (int), incremented on each page-change operation
- When counter >= threshold, save PDF to `%TEMP%/<hash>.unicolbak`
- On startup: `_check_crash_recovery()` scans temp dir for `.unicolbak` files

### 4. Menu & Settings (`app/menu_bar.py`)

- 设置 → 历史操作上限: QInputDialog for integer (default 20)
- 设置 → 自动保存阈值: QInputDialog for integer (default 10)
- 设置 → 自动保存路径: path selector + "打开文件夹" button
- Ctrl+Z/Y shortcuts added

### 5. Export Fix (`app/dialogs/export_dialog.py`)

- Select directory → input project name → save as `<dir>/<project_name>.pdf`
- No sub-folder creation

### 6. Crash Recovery

- Auto-save file: `%TEMP%/unicolpdf_autosave_<timestamp>.unicolbak`
- On startup: check for `.unicolbak` in temp dir
- If found: show dialog "检测到上次未保存的编辑，是否恢复？"
- If yes: load from `.unicolbak` (rename to `.pdf` internally or handle directly)
- Recovery file is treated as a new PDF open

## Risk Assessment

- History snapshots use `BytesIO` — memory grows with PDF size × max_steps; mitigated by configurable limit
- Undo/redo restores full PDF state — fast but replaces current doc in memory
- Crash recovery uses non-standard extension to avoid false positives
